# Contributing

Thank you for your interest in the Sparked FHIR Server Configuration.

## How to Contribute

1. **Open an issue** describing the change you'd like to make
2. **Fork the repository** and create a feature branch
3. **Submit a pull request** targeting the `main` branch

## Review Requirements

- All pull requests require review from an AEHRC team member
- Changes to GitHub Actions workflows and Terraform configuration require additional review as defined in [CODEOWNERS](.github/CODEOWNERS)
- Ensure your changes pass the automated validation checks

## Important Notes

- Do not commit secrets, credentials, or sensitive configuration values
- Use placeholder values in committed files; real values belong in gitignored `.tfvars` files or GitHub Secrets/Variables
- Follow existing patterns and conventions in the codebase

## Code of Conduct

Please be respectful and constructive in all interactions. We are committed to providing a welcoming and inclusive experience for everyone.
